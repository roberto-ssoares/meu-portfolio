# Fun√ß√£o para testar a estacionaridade
def fn_testa_estacionaridade(serie, window = 12, title='Estat√≠sticas M√≥veis e Teste Dickey-Fuller'):
    """
    Fun√ß√£o para testar a estacionaridade de uma s√©rie temporal.
    
    Par√¢metros:
    - serie: pandas.Series. S√©rie temporal a ser testada.
    - window: int. Janela para c√°lculo das estat√≠sticas m√≥veis.
    - title: str. T√≠tulo para os gr√°ficos.
    """
    # Calcula estat√≠sticas m√≥veis
    rolmean = serie.rolling(window = window).mean()
    rolstd = serie.rolling(window = window).std()

    # Plot das estat√≠sticas m√≥veis
    plt.figure(figsize = (15, 6))
    plt.plot(serie, color = 'blue', label = 'Original')
    plt.plot(rolmean, color = 'red', label = 'M√©dia M√≥vel')
    plt.plot(rolstd, color = 'black', label = 'Desvio Padr√£o M√≥vel')
    plt.legend(loc='best')
    plt.title(title)
    plt.show(block=False)
    
    # Teste Dickey-Fuller
    print('\nResultado do Teste Dickey-Fuller:')
    dfteste = adfuller(serie, autolag='AIC')
    dfsaida = pd.Series(dfteste[0:4], index = ['Estat√≠stica do Teste', 
                                               'Valor-p', 
                                               'N√∫mero de Lags Consideradas', 
                                               'N√∫mero de Observa√ß√µes Usadas'])
    for key, value in dfteste[4].items():
        dfsaida['Valor Cr√≠tico (%s)' % key] = value
        
    print(dfsaida)
    
    # Conclus√£o baseada no valor-p
    if dfsaida['Valor-p'] > 0.05:
        print('\nConclus√£o:\nO valor-p √© maior que 0.05 e, portanto, n√£o temos evid√™ncias para rejeitar a hip√≥tese nula.\nEssa s√©rie provavelmente n√£o √© estacion√°ria.')
    else:
        print('\nConclus√£o:\nO valor-p √© menor que 0.05 e, portanto, temos evid√™ncias para rejeitar a hip√≥tese nula.\nEssa s√©rie provavelmente √© estacion√°ria.')
fn_testa_estacionaridade(dados['usuarios_ativos_log'])

‚öôÔ∏è 1.3. Transforma√ß√£o Box-Cox

üìå Observe:

  • A transforma√ß√£o de raiz quadrada e a transforma√ß√£o de log pertencem a uma classe de transforma√ß√µes denominadas transforma√ß√µes de pot√™ncia.
  • A transforma√ß√£o Box-Cox √© um m√©todo de transforma√ß√£o de dados configur√°vel que suporta a raiz quadrada e a transforma√ß√£o de log, bem como um conjunto de transforma√ß√µes relacionadas.
  • Mais do que isso, pode ser configurado para avaliar um conjunto de transforma√ß√µes automaticamente e selecionar o melhor ajuste.
    • A s√©rie resultante pode ser mais linear e a distribui√ß√£o resultante mais gaussiana ou uniforme, dependendo do processo que a gerou.
  • A biblioteca scipy.stats fornece uma implementa√ß√£o da transforma√ß√£o Box-Cox.
    • A fun√ß√£o boxcox() usa um argumento, chamado lambda, que controla o tipo de transforma√ß√£o a ser executada.
  • Abaixo est√£o alguns valores comuns para lambda
    • lambda = -1. √© uma transforma√ß√£o rec√≠proca.
    • lambda = -0,5 √© uma transforma√ß√£o de raiz quadrada rec√≠proca.
    • lambda = 0,0 √© uma transforma√ß√£o de log.
    • lambda = 0,5 √© uma transforma√ß√£o de raiz quadrada.
    • lambda = 1.0 n√£o √© transforma√ß√£o.

üìå Observe:

  • A transforma√ß√£o de Box-Cox em Python
    • √â especificamente o lambda √≥timo $\lambda$ valor, √© tratado principalmente usando a scipy.stats.boxcoxfun√ß√£o.
    • Entendendo Lambda na Transforma√ß√£o de Box-Cox:
      • A transforma√ß√£o de Box-Cox √© uma transforma√ß√£o de pot√™ncia usada para estabilizar a vari√¢ncia e fazer com que os dados se assemelhem mais a uma distribui√ß√£o normal.
      • A transforma√ß√£o √© parametrizada por uma lambda $\lambda$ valor, que normalmente varia de -5 a 5.
      • Diferente $\lambda$ Os valores correspondem a diferentes tipos de transforma√ß√µes (por exemplo, $\lambda$ = 0 corresponde a um logaritmo natural, $\lambda$ =2 (para elevar os dados ao quadrado).
      • O objetivo √© encontrar o ideal $\lambda$ que melhor normaliza os dados, geralmente obtido por meio da estimativa de m√°xima verossimilhan√ßa.
  • Considera√ß√µes importantes:
    • Dados positivos: A transforma√ß√£o de Box-Cox exige que todos os pontos de dados sejam estritamente positivos.
    • Se seus dados contiverem zeros ou valores negativos, voc√™ deve desloc√°-los (por exemplo, adicionando uma constante) antes de aplicar a transforma√ß√£o.
  • Interpreta√ß√£o:
    • A transforma√ß√£o altera a escala dos dados, de modo que a interpretabilidade das unidades originais se perde, a menos que a transforma√ß√£o seja revertida.
    • Estimativa de M√°xima Verossimilhan√ßa:
      • Quando lambda n√£o √© especificado em boxcox, a fun√ß√£o usa a estimativa de m√°xima verossimilhan√ßa para encontrar o $\lambda$.
      • Valor que maximiza a fun√ß√£o de log-verossimilhan√ßa dos dados transformados, assumindo que seguem uma distribui√ß√£o normal.
